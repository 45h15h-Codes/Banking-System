# BankCore System - Phase 0 Progress

Welcome to the BankCore System repository! This README outlines our progress, core architectural decisions, and the standards we've established to ensure a scalable, secure, and maintainable microservices platform.

## Overview

We have successfully completed Phase 0 of the BankCore implementation. Our current progress includes:
- Establishing a monorepo architecture.
- Scaffolding 7 independent Laravel microservices in the `services/` directory.
- Configuring Docker-based infrastructure (`docker-compose`) and resolving initial database connectivity issues across MySQL instances.
- Standardizing the `Dockerfile.template` for building robust container images.

---

## 🏗 Architecture & Design

"Simplicity is the ultimate sophistication." Our architecture starts simple and adds complexity only when proven necessary.

### Key Architectural Principles
- **Requirements Drive Architecture**: We evaluate trade-offs before introducing new patterns. 
- **Decoupled Services**: Each Laravel service is independent and owns its data.
- **Dockerized Infrastructure**: All services run in isolated containers for consistency across environments.

### Architecture Decision Records (ADRs)
We use ADRs to capture the context and rationale behind significant technical decisions. All design records are kept in `docs/adr/`.
* **ADR-0001**: Use of Monorepo for unified service management.
* **ADR-0002**: Adoption of Docker & MySQL per service to ensure high cohesion and low coupling.
* *Note: Before proposing major changes to the infrastructure or services, please submit an RFC or an ADR for team review.*

## 📚 API Documentation

Clear and comprehensive API documentation is critical for the success of our independently deployed microservices. 

### Documentation Strategy
- **OpenAPI/Swagger**: All services must expose an OpenAPI specification.
- **Interactive Docs**: We use Swagger UI and Redoc to provide interactive try-it functionality.
- **Consistency**: Every endpoint is required to have:
  - Expected Request path, headers (Bearer Authentication), and body schema.
  - Complete Response specification, including standard and error status codes.
  - Clear usage guidelines and code examples (cURL, Javascript, etc.).

*To view the API documentation for a specific service, navigate to its respective documentation portal generated by our API Documenter tools.*

## 📊 Analytics & Product Metrics

In God we trust. All others must bring data. 
To measure the success of the BankCore system, tracking and telemetry have been ingrained from Phase 0.

### North Star Metric
- **Weekly Active Transactors (WAT)**: The number of unique users completing at least one successful financial transaction (deposit, withdrawal, transfer) within a 7-day period.

### Core Events Tracked
We track essential banking events systematically:
- `user_signed_up`: Account creation metrics.
- `onboarding_completed`: KYC and setup processes.
- `transaction_started`: User intent to perform financial movements.
- `transaction_completed`: Successful clearing of the movement.

We use cohort retention analysis and funnel monitoring to identify drop-offs occurring during user onboarding or complex financial workflows.
